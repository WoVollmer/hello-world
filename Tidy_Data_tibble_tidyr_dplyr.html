Short ‘theoretical’ consideration of tidy data

Tidy Data - What does tidy data mean ?

For detailed descriptions see

  • Hadley Wickham - Tidy Data Journal of Statistical Software

Importance of Data Tidying:

  • data tidying: structuring datasets to facilitate analysis
    • often said: 80% of data analysis is spent on the cleaning and preparing data

Statistical datasets are mostly:

  • datasets / data frames made up of rows and columns
    • columns are almost always labeled
    • rows are sometimes labeled

Tidy Data definition:

  • tidy data characteristics:
    • each variable forms a column
    • each observation forms a row
    • each type of observational unit forms a table
  • approach fits to Database normalization and Codd’s 3NF - Third normal form
    • in statistics we have focus on single dataset rather than many connected datasets in relational databases

#  - drop the features that used to be convenient but are now frustrating

Variables (columns) and Observations/Cases (rows) tidying by tidyr / dplyr functions

Each variable forms a column

  • Separate “multiple variable” column in separate columns
  • Select out / delete columns with “redundant” information
  • “Piping” with pipe operator %>% makes code simple to read

#
# "tidyr" - gather/spread function, change data frame wide/long form

# "dplyr" - grammar of data manipulation

#           group_by

#
# separate(): separates single character column into multiple columns
# select():   selects the variables/columns
#             Positive values select variables; negative values drop variables


fort_data_tbl_update <-

#
#
# gather():   gather/melt multiple columns into rows, converts data from wide 
#             form to long form
#             key:   variable name of new variable created for the gathered column names
#             value: variable name of new variable created from the cell values 
#                    of gathered columns
#             ...:   specifying range of variables/columns to be gathered, 
#                    default: all variables are selected, negative values for dropping
# mutate():   compute new column(s)
# arrange():  order rows by values of a column (low to high)
# rename():   renames column(s)


fort_general_data_tbl <- 
    fort_data_tbl_update %>%
    select(name, birthday, sex, residence, role, time_public, time_individual) %>% 
    gather(key = transport_mode, value = "avg_travel_time",
           time_public:time_individual) %>%
    mutate(transport_mode = gsub("time_", "", transport_mode)) %>% 
    #  replaces in new column 'transport_mode'
    #     time_public/_indiviudal by public/individual
    arrange(name, transport_mode)  
    # sort by name and then by transport_mode 

# note: columns are ordered with ordering within select()
#       rule: fixed values come first, followed by measured variables

Some ‘basic’ data manipulation examples with data set mtcars

Grouping: dplyr functions will manipulate each “group” separately and then combine the results

Summariy functions: take vectors as input and return single values as output

# mtcars_df: mtcars with rownames and reduced variables, but still data.frame
mtcars_df <- mtcars %>%                      # note: mtcars_df still no tibble
  rownames_to_column(var = "model") %>%      # w/ rownames
  select(model:hp, am)                       # reduce number of variables


mtcars_df %>% 
  group_by(cyl, am) %>%      # grouping first by cyl, then by am 
  # creates a "grouped" copy of a table. dplyr functions will manipulate 
  # each "group" separately and then combine the results
  summarise(cases = n(),  
            mean_mpg = mean(mpg),
            mean_hp = mean(hp)) %>%             
  # summarize: take vectors as input and return single values as output
  # grouped data: output will have one row for each group
  mutate(ratio_hp_mpg = mean_mpg / mean_hp) %>% 
  # Compute new column(s)
  arrange(mean_hp)
